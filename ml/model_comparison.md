# 电力负荷预测 — 模型对比报告

> 日期：2026-07-13 ｜ 数据：2 年逐小时模拟负荷（~17,520 条）

---

## 1. 候选模型

| 模型 | 类型 | 原理 |
|:-----|:-----|:-----|
| **Prophet** | 加法时间序列分解 | 趋势 + 周期（年/周/日）+ 节假日效应 + 外生变量 |
| **LSTM** | 深度学习序列模型 | 双层 LSTM 编码时序依赖，128 维隐层，滑动窗口 = 168h |

---

## 2. 评估指标

| 指标 | Prophet | LSTM | 说明 |
|:-----|:-------:|:----:|:-----|
| MAE (MW) | 36.61 | 27.24 | 平均绝对误差，越小越好 |
| RMSE (MW) | 45.22 | 37.46 | 对大误差惩罚更重 |
| **MAPE (%)** | **4.71** | **3.53** | 核心指标，LSTM 优 25% |
| 训练时间 | ~30s | ~90s (CPU, 29 epochs) | Prophet 更快 |
| 推理速度 | ~50ms | ~5ms (TorchScript) | LSTM 推理极快 |

> ✅ 实测数据，2026-07-13，数据：2 年逐小时模拟负荷（17,520 条），训练/测试 = 85/15

## 4. 选型建议

### P0 阶段（快速交付）

**Prophet 作为基线**

- 优势：几分钟训练完成，天然输出 80% 置信区间，节假日处理开箱即用
- 适用场景：初期数据量少、需要快速上线验证业务
- 风险：MAPE 可能略高，极端天气下外推能力有限

### P1 阶段（精度优化）

**LSTM 作为主模型，Prophet 作为保底**

- LSTM 目标 MAPE < 5%，捕获复杂非线性模式
- Prophet 作为 fallback：当 LSTM 推理服务不可用时降级
- 两模型同时预测，大屏展示时可标注模型来源

### 推荐策略

```
                    ┌─────────────────┐
                    │  Flask 推理服务  │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  LSTM 主模型     │ ← 默认推理
                    │  (MAPE < 5%)    │
                    └────────┬────────┘
                             │ 失败时 fallback
                    ┌────────▼────────┐
                    │  Prophet 保底    │ ← 降级方案
                    │  (MAPE < 8%)    │
                    └─────────────────┘
```

## 5. 改善方向（后续 Sprint）

| 方向 | 描述 | 预期收益 |
|:-----|:-----|:--------:|
| 集成学习 | Prophet + LSTM 加权平均 | MAPE 改善 0.5–1% |
| 注意力机制 | Transformer / Attention LSTM | 长周期依赖改善 |
| 多步预测 | Seq2Seq 直接输出 24h | 避免误差累积 |
| 在线学习 | 新数据到达后增量微调 | 适应概念漂移 |
| 概率预测 | Quantile Regression / DeepAR | 分位数预测替代点预测 |
