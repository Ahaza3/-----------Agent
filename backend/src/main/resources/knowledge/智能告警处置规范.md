# 智能告警处置规范
> 分类：ALERT_HANDLING

## 告警分级
告警等级应依据规则配置、偏差程度、持续时间和影响范围确定。红色、橙色、黄色等等级的具体含义必须以系统当前规则和项目约定为准，不能仅凭颜色推断真实运行风险。

## 标准处置流程
建议流程为：查看告警详情、核对触发时间和当前值、确认数据来源、分析影响范围、提出处置建议、由调度或运维人员人工确认、记录工单动作、跟踪恢复状态。系统可以生成建议和工单草稿，但不应绕过人工确认自动提交正式工单。

## 工单状态
告警已读、告警已确认、工单已解决和工单已关闭是不同状态。已读只代表查看过信息；已确认代表责任人员接收告警；已解决代表问题处理完成；已关闭代表流程完成并完成记录归档。

## 回复要求
智能助手应说明判断依据、数据来源和不确定性。缺少测量值、设备状态或现场信息时，应明确说明无法判断具体故障原因，不得编造跳闸、故障或备用不足等事实。
