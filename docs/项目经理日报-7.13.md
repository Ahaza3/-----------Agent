## 今天遇到的问题及解决过程描述
1.问题：前后端接口规范未对齐，两端独立搭建框架时，接口请求格式、返回字段、命名风格不统一，后续联调会出现大量兼容问题。
解决方法：组织前后端成员共同制定接口规范文档，约定统一采用 RESTful 风格接口，固定通用返回体格式（状态码、消息、数据字段），统一参数命名规范，先输出接口契约再推进业务开发，提前规避联调冲突。
2.问题：GitHub 多人协作时分支管理混乱，多人直接向主分支提交代码，频繁出现代码合并冲突，且提交记录杂乱无章。
解决方法：制定团队 Git 协作规范，划分主分支、公共开发分支与个人功能分支，要求所有功能开发在个人分支完成；提交前先拉取最新开发分支代码解决本地冲突，再提交合并申请由仓库管理员审核，同时统一 commit 提交的命名格式。
