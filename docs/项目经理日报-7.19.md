## 今天遇到的问题及解决过程描述
解决方法（Flyway 建表失败）：
1. check_db.py 诊断发现 Flyway V2 迁移记录残留但表未创建，确认为之前迁移执行中断导致。
2. fix_tables.py 手动执行 CREATE TABLE IF NOT EXISTS 补齐 5 张缺失表（prediction_result / alert_event / alert_rule / model_version / conversation），结构与 V1__init_schema.sql 一致。
3. 清理 flyway_schema_history 中失败的 V2 记录，设置 baseline-on-migrate=true 确保后续迁移正常执行。
4. check_db.py 再次验证：所有表正常存在，Flyway 基线校准完成。
