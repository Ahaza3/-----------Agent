# Agent 工具：query_alert_accuracy 使用说明

## 功能概述

`query_alert_accuracy` 工具用于分析告警准确率与误报模式，帮助调度员和运维人员识别规则调优机会，减少误报干扰。

**核心能力：**
- 统计指定时间段/节点的真实告警、误报、重复告警数量
- 计算告警准确率（真实告警 / (真实告警 + 误报) × 100%）
- 识别误报的主要根因（阈值过低、数据异常、设备测试等）
- 自动生成规则调优建议

## 适用场景

### 场景 1：全局告警质量评估
**用户提问：**
> "最近 30 天告警准确率怎么样？"

**Agent 行为：**
- 调用 `query_alert_accuracy(days=30)`
- 返回：总告警数、真实告警、误报、准确率、误报根因 TOP5
- 生成建议：如"多次因'阈值过低'误报，建议分析历史负荷峰值并适当上调阈值"

### 场景 2：节点级误报分析
**用户提问：**
> "FEED-01 这个馈线最近误报多吗？"

**Agent 行为：**
- 调用 `query_alert_accuracy(nodeId=3, days=30)`（假设 FEED-01 的 nodeId=3）
- 返回该节点专属的准确率和误报根因
- 判断：若误报率 > 50% 且根因集中在 `THRESHOLD_TOO_LOW`，则建议上调该节点阈值

### 场景 3：按级别筛选
**用户提问：**
> "红色告警误报率高吗？"

**Agent 行为：**
- 调用 `query_alert_accuracy(level='RED', days=30)`
- 只统计红色级别告警，排除橙色/黄色干扰
- 输出红色告警的准确率和误报根因分布

### 场景 4：长周期趋势
**用户提问：**
> "过去 90 天告警质量有改善吗？"

**Agent 行为：**
- 分别调用 `query_alert_accuracy(days=90)` 和 `query_alert_accuracy(days=30)`
- 对比两个时间段的准确率变化
- 判断趋势并说明是改善还是恶化

## 工具参数

| 参数 | 类型 | 必填 | 说明 | 示例 |
|:-----|:-----|:----:|:-----|:-----|
| `nodeId` | integer | 否 | 拓扑节点ID，用于分析特定节点 | `3` |
| `days` | integer | 否 | 统计时间范围（天），默认30天，最大90天 | `30` |
| `level` | string | 否 | 告警级别筛选：RED/ORANGE/YELLOW | `"RED"` |

## 自动生成的建议规则

工具会根据反馈数据自动生成以下建议：

| 触发条件 | 建议内容 |
|:---------|:---------|
| 误报数 > 真实告警 且 误报 > 5 | "误报数量超过真实告警，建议重点核查告警规则配置" |
| `THRESHOLD_TOO_LOW` 根因 ≥ 3 次 | "多次因'阈值过低'误报，建议分析历史负荷峰值并适当上调阈值" |
| `DATA_ANOMALY` 根因 ≥ 3 次 | "多次因'数据异常'误报，建议核查遥测数据质量或增加数据校验逻辑" |
| 重复告警 > 5 条 | "存在较多重复告警工单，建议优化告警去重或合并逻辑" |

## 与现有反馈流程的集成

此工具基于 **P0-07 工单反馈与告警准确率** 功能（已实现）：

1. **数据来源**：工单关闭时，调度员提交的结构化反馈（`ticket_feedback` 表）
2. **反馈字段**：
   - `alertClassification`: TRUE_ALERT | FALSE_POSITIVE | DUPLICATE | TEST_ALERT | PREWARNING_ONLY | INCONCLUSIVE
   - `rootCauseCode`: THRESHOLD_TOO_LOW | DATA_ANOMALY | EQUIPMENT_TEST | MAINTENANCE_WINDOW 等
3. **统计范围**：只统计 `CLOSED` 状态工单，`LEGACY_MISSING` 反馈单独计数

## 权限控制

**允许角色**：`DISPATCHER`、`OPERATOR`、`SYSTEM_ADMIN`

普通用户无法调用此工具，防止越权查看运营数据。

## 注意事项

1. **准确率计算仅统计已关闭且有反馈的工单**，历史遗留工单（无反馈）不参与计算
2. **建议仅供参考**，最终规则调整需人工分析历史负荷曲线、业务特性和安全裕度
3. **模拟数据环境**：当前数据源为模拟负荷，准确率结果不代表真实电网的告警质量
4. **不自动修改规则**：工具只读分析，不会自动调整阈值或修改规则配置

## 后续增强方向

可结合此工具进一步优化：

- **趋势图表**：前端按月展示准确率变化趋势
- **节点排行**：自动识别误报率最高的 TOP 10 节点
- **规则草稿生成**：基于历史负荷分位数自动生成阈值调整草稿（需人工审核）
- **A/B 测试**：对比调整前后准确率变化，验证规则优化效果
- **多维交叉分析**：按时段（0-6时 vs 12-18时）、天气条件、节点类型细分准确率
