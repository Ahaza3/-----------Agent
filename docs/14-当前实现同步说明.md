## 2. 已完成的核心功能

### 2.1 大屏与数据

- 历史负荷、恢复段、实时模拟负荷分段展示。
- 恢复段与实时段之间使用前端平滑 bridge 曲线连接。
- bridge 点不写入数据库，不影响后续真实整点数据归档。
- 页面展示数据来源，区分历史数据、恢复模拟数据和实时模拟数据。
- 实时负荷当前仍是演示用模拟数据，不代表真实 SCADA 数据。

### 2.2 Agent 聊天

- 支持 SSE 流式事件：`thinking`、`text`、`chart`、`done`、`error`。
- 助手 Markdown 文本以完整消息发送，前端在流结束前后都能正确渲染。
- ECharts option 保存到 `conversation.chart_option`，历史会话重新打开可恢复图表。
- `done` 事件包含数据来源、时间范围、模型版本和模拟数据标识等可信度信息。

### 2.3 告警与工单

- 告警文案使用固定模板，保证低延迟；工单详情的 AI 智能研判真实调用 LLM，失败时降级为规则研判。
- 告警生命周期已支持恢复时间、确认时间和确认人。
- 已读与人工确认处置分开记录。
- 红色告警只生成“待确认工单草稿”，不会自动提交正式工单。
- 调度员确认后才创建正式 `alert_ticket`。
- 未创建工单的告警不显示处置时间线，也不会复用其他工单的 actions。
- 告警研判接口：
  - `GET /api/v1/alert/events/{id}/judgement`：只读已有缓存，没有缓存时返回空。
  - `POST /api/v1/alert/events/{id}/judgement`：显式重新调用研判。

### 2.4 模型管理与重训练

- 模型版本列表是纯查询，不会隐式同步或清理本地模型产物。
- `POST /api/v1/model/versions/sync` 专门负责同步本地模型产物。
- 模型页面展示数据库发布版本、Flask 实际运行模型及“已生效/待重启/推理服务异常”状态。
- 重训练任务会记录数据时间范围、样本数、耗时、失败原因和产物清单。
- 启动重训练前，会从 MySQL 的 `load_data` 导出最新负荷和天气字段，覆盖生成 `featured_load_data.csv`，再调用训练脚本。
- LSTM 重训练会同步生成模型、输入 scaler、目标 scaler、特征列和元信息，并在登记前校验产物完整性。
- 产品已决定后续不再使用 Prophet。当前代码仍保留 Prophet 后端兼容能力和模型页入口，待下一步移除页面重训练按钮；历史 Prophet 版本和旧模型文件暂时保留，避免破坏历史数据。
- 版本切换或重训练完成后不会自动热重载 Flask，需要根据页面状态重启 Flask 服务。

### 2.5 预测运营与天气

- 支持预测质量摘要、线上预测复盘、滚动 MAPE/RMSE/MAE、峰值误差。
- 有验证集 RMSE 时生成运行参考区间，标注为参考区间，不宣称严格概率置信区间。
- 已接入未来天气预报缓存：`weather_forecast` 表、同步任务、查询接口和模型推理输入。
- 新训练的 LSTM 可使用未来 24 小时温度和湿度；旧模型仍可兼容，天气不可用时回退历史天气。

## 3. 关键接口口径

| 功能 | 接口 | 当前行为 |
|:-----|:-----|:---------|
| 模型版本查询 | `GET /api/v1/model/versions` | 纯查询 |
| 本地模型同步 | `POST /api/v1/model/versions/sync` | 扫描并登记本地产物 |
| 模型激活 | `PUT /api/v1/model/versions/{id}/activate` | 更新数据库发布版本，不自动重启 Flask |
| LSTM 重训练 | `POST /api/v1/model/retrain` | 导出 MySQL 特征数据后启动训练 |
| 重训练状态 | `GET /api/v1/model/retrain/status` | 查询当前任务 |
| 重训练历史 | `GET /api/v1/model/retrain/history` | 查询持久化训练记录 |
| 缓存研判查询 | `GET /api/v1/alert/events/{id}/judgement` | 只读缓存，无缓存返回空 |
| 重新研判 | `POST /api/v1/alert/events/{id}/judgement` | 显式调用 LLM 或规则降级 |
| 未来天气查询 | `GET /api/v1/weather/forecast` | 查询缓存预报 |
| 未来天气同步 | `POST /api/v1/weather/sync` | 更新预报快照 |

## 4. 当前边界与待办

- 实时负荷仍是演示模拟数据，尚未接入真实 SCADA 或生产数据源。
- Flask 当前按启动时加载模型，数据库发布版本与运行模型需要通过页面状态和服务重启确认一致。
- Prophet 仅作为历史兼容项，后续可在确认无历史依赖后清理训练脚本、产物和依赖。
- 训练任务当前已有数据库记录，但后台执行仍依赖本地进程，不是分布式任务队列。
- 预测参考区间是基于验证集 RMSE 的运行参考，不等同于严格概率预测区间。
