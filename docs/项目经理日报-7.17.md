## 今天遇到的问题及解决过程描述
问题一的解决方法：
1.修改后端与 Flask 的配置文件，将数据库 host 改为 docker-compose 中定义的 MySQL 服务名，实现容器内网互通。
2.在 docker-compose 中为后端、Flask 服务添加 depends_on 与健康检查配置，强制等待 MySQL 初始化完成后再启动应用服务。
3.补充启动脚本，服务启动前先执行数据库连通性检测，失败则重试等待。

问题二的解决方法：
1.后端接口增加参数格式兼容逻辑，自动将 / 格式的日期统一转换为标准格式，同时补充参数校验与友好错误提示。
2.在 Service 层增加空值兜底处理，缺失数据自动填充为 0 或插值结果，保证序列化正常。
3.前端统一请求参数格式，新增接口异常捕获与加载失败兜底展示。
