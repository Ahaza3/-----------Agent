## 1. 设计约定

### 1.1 基础规范

| 项目 | 规范 |
|:-----|:-----|
| 基础路径 | `/api/v1/` |
| 请求格式 | `application/json; charset=utf-8` |
| 响应格式 | `application/json; charset=utf-8` |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601（`yyyy-MM-ddTHH:mm:ss`） |
| 分页参数 | `page`（从 1 开始）、`size`（默认 20，最大 100） |
| 认证方式 | JWT `Authorization: Bearer <token>`（P2 阶段启用） |

### 1.2 统一响应封装 `R<T>`

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": 1704067200000
}
```

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| code | int | 0 = 成功，非 0 = 失败（见错误码表） |
| message | string | 提示信息 |
| data | T \| null | 业务数据（泛型） |
| timestamp | long | 响应时间戳（毫秒） |

### 1.3 错误码定义

| 错误码 | HTTP 状态码 | 说明 |
|:------:|:-----------|:-----|
| 0 | 200 | 成功 |
| 400 | 400 | 请求参数校验失败 |
| 401 | 401 | 未认证 / Token 过期 |
| 403 | 403 | 无权限 |
| 404 | 404 | 资源不存在 |
| 500 | 500 | 服务器内部错误 |
| 1001 | 200 | 预测模型加载失败 |
| 1002 | 200 | Agent 调用 LLM 超时 |
| 1003 | 200 | 数据源连接异常 |

### 1.4 分页响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [],
    "total": 17520,
    "page": 1,
    "size": 20,
    "pages": 876
  },
  "timestamp": 1704067200000
}
```

## 3. 数据查询接口

### 3.1 时间范围查询负荷

```
GET /api/v1/data/load?start={iso_time}&end={iso_time}
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| start | string | ✅ | 起始时间 ISO 8601（含） |
| end | string | ✅ | 结束时间 ISO 8601（不含） |

**请求示例**：
```
GET /api/v1/data/load?start=2024-01-01T00:00:00&end=2024-01-02T00:00:00
```

**成功响应** (200)：
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "time": "2024-01-01T00:00:00",
      "loadMw": 850.3,
      "temperature": 5.2,
      "humidity": 72.0,
      "isHoliday": false,
      "hour": 0,
      "dayOfWeek": 0,
      "month": 1
    },
    {
      "time": "2024-01-01T01:00:00",
      "loadMw": 820.1,
      "temperature": 4.8,
      "humidity": 74.5,
      "isHoliday": false,
      "hour": 1,
      "dayOfWeek": 0,
      "month": 1
    }
  ],
  "timestamp": 1704067200000
}
```

**错误响应** (400)：
```json
{
  "code": 400,
  "message": "参数校验失败：start 不能晚于 end",
  "data": null,
  "timestamp": 1704067200000
}
```

**说明**：
- 单次查询最大跨度为 365 天
- 数据按 `time` 升序排列
- 每小时一条记录，24 条 / 天

### 5.3 告警规则管理（P1）

```
GET    /api/v1/alert/rules          # 查询所有规则
POST   /api/v1/alert/rules          # 创建规则
PUT    /api/v1/alert/rules/{id}      # 更新规则
POST   /api/v1/alert/rules/{id}/snooze?minutes=30          # 暂挂规则
PUT    /api/v1/alert/rules/{id}/maintenance?enabled=true  # 维护模式
DELETE /api/v1/alert/rules/{id}      # 删除规则
```

**告警规则对象**：
```json
{
  "id": 1,
  "name": "夏季工作日高峰阈值",
  "type": "THRESHOLD",
  "config": {
    "threshold": 1200,
    "redRatio": 1.10,
    "orangeRatio": 1.00,
    "yellowRatio": 0.90,
    "coolingTime": 3600,
    "triggerDuration": 10,
    "hysteresis": 20,
    "maintenance": false
  },
  "isActive": true,
  "createdAt": "2024-01-01T00:00:00",
  "updatedAt": "2024-01-01T00:00:00"
}
```

`triggerDuration` 为连续超限触发时长（秒），`0` 表示立即触发；`hysteresis` 为恢复死区（MW），恢复条件为负荷低于黄色阈值减去该值。`maintenance=true` 或当前时间早于 `suspendUntil` 时不产生新告警。

### 5.4 告警运营指标（P1）

```
GET /api/v1/alert/events/metrics?start={ISO_DATE_TIME}&end={ISO_DATE_TIME}
```

不传时间范围时默认统计最近 7 天，返回 `total`、各级别数量、`duplicateRate`、`averageAckMinutes`、`averageRecoveryMinutes` 等指标。

## 6. Agent 对话接口

### 6.1 自然语言对话（SSE 流式）

```
POST /api/v1/agent/chat
Content-Type: application/json
Accept: text/event-stream
```

**请求体**：
```json
{
  "message": "昨天最高负荷出现在几点？比前天高了多少？",
  "conversationId": "conv_abc123"
}
```

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| message | string | ✅ | 用户自然语言消息 |
| conversationId | string | ❌ | 会话 ID，不传则创建新会话 |

**SSE 事件流响应**：

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no
```

**事件类型**：

| 事件 | 说明 | data 内容 |
|:-----|:-----|:----------|
| `thinking` | Agent 正在思考/查询 | `{"content": "正在查询昨日负荷数据..."}` |
| `text` | 文本回答片段 | `{"content": "昨日最高负荷出现在 18:30"}` |
| `chart` | ECharts 图表配置 | `{"config": {...}}` |
| `error` | 发生错误 | `{"message": "LLM 服务超时"}` |
| `done` | 流结束 | `{}` |

**完整 SSE 流示例**：
```

event: thinking
data: {"content":"正在查询昨日负荷数据..."}

event: text
data: {"content":"昨日最高负荷出现在 "}

event: text
data: {"content":"18:30，达到 1250MW，"}

event: text
data: {"content":"较前日最高值 1180MW 增长了 5.9%。"}

event: chart
data: {"config":{"xAxis":{"type":"time"},"series":[{"name":"昨日","data":[...]},{"name":"前日","data":[...]}]}}

event: done
data: {}

```

**错误 SSE 流示例**：
```

event: error
data: {"message":"LLM 服务调用超时，请稍后重试"}

event: done
data: {}

```

**对话历史**（P2 启用）：
```
GET /api/v1/agent/conversations              # 历史会话列表
GET /api/v1/agent/conversations/{id}         # 会话详情
DELETE /api/v1/agent/conversations/{id}      # 删除会话
```

## 7. WebSocket 推送

### 7.1 连接端点

```
ws://localhost:8080/ws/dashboard
```

使用 STOMP over SockJS 协议（`/ws/dashboard` 为 SockJS 端点）。

### 7.2 订阅主题

| 主题 | 说明 | 推送频率 |
|:-----|:-----|:---------|
| `/topic/load` | 实时负荷数据 | 每秒 |
| `/topic/predictions` | 预测结果更新 | 每小时 |
| `/topic/alerts` | 告警事件推送 | 事件触发 |
| `/topic/system` | 系统状态变更 | 事件触发 |

### 7.3 消息格式

**实时负荷推送** — `/topic/load`：
```json
{
  "type": "load_update",
  "data": {
    "time": "2024-01-01T10:00:00",
    "loadMw": 1100.5,
    "temperature": 12.3,
    "humidity": 58.0
  }
}
```

**告警推送** — `/topic/alerts`：
```json
{
  "type": "alert",
  "data": {
    "id": 42,
    "triggerTime": "2024-01-01T18:30:00",
    "level": "RED",
    "currentValue": 1320.5,
    "thresholdValue": 1200.0,
    "aiAnalysis": "⚠️ 紧急告警：当前负荷 1321MW..."
  }
}
```

**前端 STOMP 订阅示例**：
```typescript
const client = new Client({
  brokerURL: 'ws://localhost:8080/ws/dashboard',
  onConnect: () => {
    client.subscribe('/topic/load', (msg) => {
      const data = JSON.parse(msg.body);
      store.updateLoad(data.data);
    });
    client.subscribe('/topic/alerts', (msg) => {
      const data = JSON.parse(msg.body);
      store.addAlert(data.data);
    });
  }
});
client.activate();
```

### 8.2 模型版本管理

```
GET    /api/v1/model/versions                    # 模型版本列表
POST   /api/v1/model/versions/sync               # 扫描并登记 ml/models 下的本地模型产物
PUT    /api/v1/model/versions/{id}/activate      # 激活指定版本（也用于回滚）
POST   /api/v1/model/retrain                     # 手动触发后台重训练，body: {"modelName":"LSTM|PROPHET"}
GET    /api/v1/model/retrain/status              # 查询最近一次重训练任务状态
GET    /api/v1/model/retrain/history             # 查询最近 20 条训练记录
```

模型版本管理仅对 `OPERATOR` 和 `SYSTEM_ADMIN` 开放。列表接口会自动登记 `ml/models/lstm_scripted.pt` 与 `ml/models/prophet_model.pkl`，也可手动调用同步接口。激活操作会在事务内将其他版本置为非激活，再将目标版本设为当前版本；当前系统的 Flask 推理服务仍按自身配置加载模型文件，版本切换和重训练完成都不会自动触发 Flask 热重载。

训练记录保存数据起止时间、样本数、训练耗时、输出尾部和产物清单。LSTM 训练成功前会检查模型权重、TorchScript、元信息、输入/目标 scaler 和特征列文件是否齐全。

## 附录 B：当前实现同步补充（2026-07-19）

### B.1 Agent SSE 事件格式

`POST /api/v1/agent/chat` 当前使用 SSE 返回以下事件：

| 事件 | 数据格式 | 说明 |
|:-----|:---------|:-----|
| `thinking` | `{ "content": "正在查询..." }` | 中间状态，可出现多次 |
| `text` | `{ "content": "完整 Markdown 文本" }` | 一次性发送完整文本，保留换行 |
| `chart` | `{ "option": { ... } }` | ECharts option |
| `done` | `{ "conversationId": "...", "provenance": { ... } }` | 流结束，同时返回数据来源和模型可信度信息 |
| `error` | `{ "message": "..." }` | 异常信息 |

历史会话详情接口 `GET /api/v1/agent/conversations/{id}` 的消息对象补充字段：

```json
{
  "id": 1,
  "role": "assistant",
  "content": "## 分析结果\n...",
  "chartOption": "{\"xAxis\":...,\"series\":[...]}",
  "createdAt": "2026-07-19T10:00:00"
}
```

`chartOption` 为字符串化 JSON，前端解析后恢复图表。用户消息通常为 `null`。

### B.2 告警智能研判接口

```http
GET /api/v1/alert/events/{id}/judgement
POST /api/v1/alert/events/{id}/judgement
```

返回示例：

```json
{
  "alertId": 1001,
  "level": "RED",
  "currentLoad": 1240.0,
  "thresholdValue": 1100.0,
  "trendDirection": "RISING",
  "forecastPeakLoad": 1288.0,
  "hasExistingTicket": false,
  "hasOpenSimilarTicket": false,
  "shouldCreateTicket": true,
  "autoCreateTicket": false,
  "recommendedPriority": "URGENT",
  "decisionReason": "LLM 生成的研判原因",
  "dispatcherAdvice": "调度员处置建议",
  "operatorAdvice": "运维核查建议",
  "source": "LLM_AGENT",
  "createdAt": "2026-07-19T10:00:00"
}
```

说明：

- `source = LLM_AGENT` 表示详情研判真实调用 LLM。
- `source = RULE_BASED_FALLBACK` 表示 LLM 失败后使用规则降级。
- 红色告警只建议建单，`autoCreateTicket = false`，正式建单需要调度员确认。

### B.3 告警工单确认流程

```http
POST /api/v1/alerts/{alertId}/ticket
Content-Type: application/json

{
  "summary": "调度员确认后的工单概要"
}
```

前端红色告警弹窗中的“待确认工单草稿”不对应数据库草稿状态，只有调用上述接口成功后才创建正式 `alert_ticket` 记录。
