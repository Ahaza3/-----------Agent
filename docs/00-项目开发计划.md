## 🚫 AI 禁飞区规则

### 什么是 AI 禁飞区？

本项目划定 **3 个核心功能** 为「AI 禁飞区」——**AI 只许看思路，不许直接生成代码**。

### 为什么设禁飞区？

AI 编程工具生成代码效率极高，但学生容易沦为"代码搬运工"，完全不理解底层逻辑。禁飞区机制迫使每位成员真正掌握核心技术。

### 本项目 3 个禁飞区

| 编号 | 禁飞区模块 | 对应文件 | 验收方式 |
|:-----|:-----------|:---------|:---------|
| 🚫 **NFZ-1** | Agent 核心循环 | `AgentCore.java`（~150 行） | 逐行讲解 Function Calling 编排流程 |
| 🚫 **NFZ-2** | Prompt 工程 | `buildSystemPrompt()` + Tool Definitions | 讲解 System Prompt 设计思路 & Tool Schema 定义 |
| 🚫 **NFZ-3** | 告警检测引擎 | `ThresholdDetector.java` + `AlertTemplate.java` | 讲解分级逻辑 & 模板生成策略 |

### 执行规则

- ❌ 禁飞区内代码 **禁止直接粘贴 AI 生成结果**
- ✅ **可以参考** AI 给出的解题思路、伪代码、架构建议
- ⚡ 答辩日导师 **现场随机抽查**，要求学生逐行讲解禁飞区代码
- 🔴 无法完整讲解 → **该模块分数直接清零**

## 🤖 三层 AI 协作范式

| 层级 | 模式 | 场景示例 |
|:-----|:-----|:---------|
| 💡 **启发式** | 与 AI 讨论方案，拓展思路 | "这个 Agent 架构有哪些设计模式可选？各有什么优缺点？" |
| ✍️ **精确式** | 手写基础代码，AI 辅助优化迭代 | 写出 AgentCore 骨架 → AI 建议异常处理 & 边界条件 |
| 🔒 **约束式** | 前置定义规范约束，可控 AI 输出 | "按以下接口规范生成 Controller 层代码，禁止引入额外依赖" |

> 🚫 **硬性要求**：禁止无约束 Vibe Coding！所有 AI 生成代码必须经过 Code Review 后方可合并。
