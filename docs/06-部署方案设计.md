## 1. 部署架构概述

### 1.1 服务拓扑

```
                         ┌──────────────────────────────────────┐
                         │          🌐 用户浏览器                 │
                         │     http://localhost                  │
                         └──────────┬───────────────────────────┘
                                    │ HTTP :80
                                    ▼
                         ┌──────────────────────┐
                         │     📡 Nginx          │
                         │     nginx:alpine      │
                         │     端口映射 80:80     │
                         └──────┬───────────────┘
                                │
                  ┌─────────────┼─────────────┐
                  │ /api/*      │ /*           │
                  │ WebSocket   │ 静态资源      │
                  ▼             ▼              │
    ┌──────────────────┐  ┌──────────────┐   │
    │  ☕ Spring Boot   │  │  ⚛️ React SPA │   │

    │  Jar 单体部署      │  │  build 产物    │   │
    └──┬───────┬───────┘  └──────────────┘   │
       │       │                              │
       ▼       ▼                              │
    ┌──────┐ ┌──────┐                         │
    │ MySQL│ │ Redis│                         │
    │ :3306│ │ :6379│                         │
    └──────┘ └──────┘                         │
       │                                       │

    ┌──────────────────┐                      │
    │  🐍 Flask 推理    │                      │

    │  ml/app.py       │                      │
    └──────────────────┘                      │
                                              │
    ┌──────────────────┐                      │
    │  🌐 LLM API       │ (外部，不走容器)      │
    │  DeepSeek / Qwen  │                      │
    └──────────────────┘
```

### 1.2 服务清单

| 服务 | 容器名 | 镜像 | 端口（宿主机:容器） | 依赖 |
|:-----|:-------|:-----|:------------------|:-----|
| Nginx | `nginx` | `nginx:alpine` | 80:80 | backend |
| Backend | `backend` | `power-load-backend:latest` | 8080:8080 | mysql, redis, ml-inference |
| Frontend | `frontend` | `power-load-frontend:latest` | (Nginx 托管) | — |
| MySQL | `mysql` | `mysql:8.0` | 3306:3306 | — |
| Redis | `redis` | `redis:7-alpine` | 6379:6379 | — |
| ML Inference | `ml-inference` | `power-load-ml:latest` | 5000:5000 | — |

### 1.3 网络架构

```
网络: power-load-net (bridge)
├── nginx          ← 对外暴露 :80
├── backend        ← 内部 :8080
├── frontend       ← Nginx 内部代理静态文件
├── mysql          ← 内部 :3306
├── redis          ← 内部 :6379
└── ml-inference   ← 内部 :5000
```

## 2. 开发环境（Day 2-12）

> ⚠️ **开发阶段不要用 Docker Compose 完整编排！** 每次改代码 rebuild 镜像太慢。

### 2.1 开发工作流

```bash
# ── 第一步：只起数据库（Docker） ──
docker run -d --name mysql-dev \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=power_load \
  mysql:8.0

docker run -d --name redis-dev \
  -p 6379:6379 \
  redis:7-alpine

# ── 第二步：本地运行服务（热重载） ──
# 终端 1：后端
cd backend && mvn spring-boot:run
# → http://localhost:8080

# 终端 2：前端（Vite HMR，秒级热更新）
cd frontend && npm run dev
# → http://localhost:5173

# 终端 3：ML 推理服务
cd ml && python app.py
# → http://localhost:5000
```

### 2.2 Spring Boot Dev 配置

```yaml
# backend/src/main/resources/application-dev.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/power_load
    username: root
    password: root
  data:
    redis:
      host: localhost
      port: 6379

ml:
  service:
    url: http://localhost:5000

llm:
  api:
    key: ${LLM_API_KEY}
    url: ${LLM_API_URL:https://api.deepseek.com/v1}
```

### 2.3 数据库初始化

```bash
# Flyway 自动在应用启动时执行迁移
# 如需手动：
cd backend
mvn flyway:migrate -Dflyway.url=jdbc:mysql://localhost:3306/power_load \
                   -Dflyway.user=root -Dflyway.password=root

# 生成模拟数据（仅首次）
cd ml
python generate_mock_data.py
```

## 3. 生产部署（Day 13-15）

### 3.1 快速部署（Docker Compose 一键启动）

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env：填入 LLM_API_KEY、MYSQL_ROOT_PASSWORD

# 2. 构建镜像 + 启动全部服务
docker-compose up -d --build

# 3. 查看日志
docker-compose logs -f backend

# 4. 验证
curl http://localhost/api/v1/system/health
# → {"code":0,"data":{"status":"UP",...}}

# 5. 停止
docker-compose down
```

### 3.2 环境要求

| 依赖 | 最低版本 | 说明 |
|:-----|:---------|:-----|
| Docker | 24.0+ | Docker Engine with Compose v2 |
| 内存 | 8 GB+ | 6 个容器运行（MySQL 占用约 500MB） |
| 磁盘 | 10 GB+ | 含镜像 + MySQL 数据卷 |
| 网络 | 可访问 LLM API | DeepSeek / 通义千问公网 API |

## 4. Docker Compose 编排

```yaml
# docker-compose.yml
version: '3.8'

services:
  # ─── 反向代理 ───
  nginx:
    image: nginx:alpine
    container_name: nginx
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - frontend-dist:/usr/share/nginx/html:ro
    depends_on:
      - backend
      - frontend
    restart: unless-stopped
    networks:
      - power-load-net

  # ─── 后端服务 ───
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: backend
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=docker
      - MYSQL_HOST=mysql
      - MYSQL_PORT=3306
      - MYSQL_DATABASE=power_load
      - MYSQL_USER=root
      - MYSQL_PASSWORD=${MYSQL_ROOT_PASSWORD}
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - ML_SERVICE_URL=http://ml-inference:5000
      - LLM_API_KEY=${LLM_API_KEY}
      - LLM_API_URL=${LLM_API_URL:-https://api.deepseek.com/v1}
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_started
      ml-inference:
        condition: service_started
    restart: unless-stopped
    networks:
      - power-load-net

  # ─── 前端构建 ───
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: frontend
    volumes:
      - frontend-dist:/app/dist
    networks:
      - power-load-net

  # ─── MySQL ───
  mysql:
    image: mysql:8.0
    container_name: mysql
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: power_load
    volumes:
      - mysql-data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-p${MYSQL_ROOT_PASSWORD}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    restart: unless-stopped
    networks:
      - power-load-net

  # ─── Redis ───
  redis:
    image: redis:7-alpine
    container_name: redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 5
    restart: unless-stopped
    networks:
      - power-load-net

  # ─── ML 推理服务 ───
  ml-inference:
    build:
      context: ./ml
      dockerfile: Dockerfile
    container_name: ml-inference
    ports:
      - "5000:5000"
    volumes:
      - ./ml/models:/app/models:ro
    healthcheck:
      test: ["CMD", "python", "-c", "import requests; requests.get('http://localhost:5000/health')"]
      interval: 30s
      timeout: 10s
      retries: 3
    restart: unless-stopped
    networks:
      - power-load-net

volumes:
  mysql-data:
  redis-data:
  frontend-dist:

networks:
  power-load-net:
    driver: bridge
```

### 4.1 启动顺序

```
1. mysql    (启动 → health check 通过)
2. redis    (启动)
3. ml-inference (启动 → 加载模型)
4. backend  (启动 → Flyway 迁移 → 连接 MySQL/Redis/ML)
5. frontend (构建 → 产出 dist/)
6. nginx    (启动 → 代理请求)
```

## 5. Dockerfile 设计

### 5.1 后端 Dockerfile（多阶段构建）

```dockerfile
# backend/Dockerfile
# ─── 阶段 1：构建 ───
FROM maven:3.9-eclipse-temurin-17-alpine AS builder
WORKDIR /app
COPY pom.xml .
# 缓存依赖层（源码没变就不用重新下载）
RUN mvn dependency:go-offline -B -q
COPY src/ src/
RUN mvn package -DskipTests -q

# ─── 阶段 2：运行 ───
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
# 创建非 root 用户
RUN addgroup -S app && adduser -S app -G app
# 复制 JAR
COPY --from=builder /app/target/*.jar app.jar
# 健康检查
HEALTHCHECK --interval=30s --timeout=5s --start-period=60s --retries=3 \
  CMD wget -qO- http://localhost:8080/actuator/health || exit 1
USER app
EXPOSE 8080
ENTRYPOINT ["java", "-Xmx512m", "-jar", "app.jar"]
```

### 5.2 前端 Dockerfile（多阶段构建）

```dockerfile
# frontend/Dockerfile
# ─── 阶段 1：构建 ───
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm run build

# ─── 阶段 2：产出 dist（由 Nginx 挂载） ───
FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/dist ./dist
# 持续运行以保持容器存活（实际由 Nginx 挂载读取）
CMD ["tail", "-f", "/dev/null"]
```

### 5.3 ML 推理 Dockerfile

```dockerfile
# ml/Dockerfile
FROM python:3.11-slim
WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt -q

# 复制模型文件和应用
COPY models/ models/
COPY app.py .

# 健康检查
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD python -c "import requests; requests.get('http://localhost:5000/health')" || exit 1

EXPOSE 5000
CMD ["python", "app.py"]
```

## 6. Nginx 配置

```nginx
# nginx.conf
events {
    worker_connections 1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer"';

    # 上游服务
    upstream backend {
        server backend:8080;
    }

    server {
        listen 80;
        server_name localhost;

        # ─── 前端静态资源 ───
        location / {
            root   /usr/share/nginx/html;
            index  index.html;
            try_files $uri $uri/ /index.html;  # SPA 路由回退
        }

        # ─── API 代理 ───
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_read_timeout 120s;  # Agent 流式响应需要较长超时
        }

        # ─── WebSocket 代理 ───
        location /ws/ {
            proxy_pass http://backend;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_read_timeout 86400s;  # WebSocket 长连接
        }

        # ─── Actuator 健康检查（仅内部） ───
        location /actuator/ {
            proxy_pass http://backend;
            # 生产环境可加 IP 白名单
        }

        # ─── Gzip 压缩 ───
        gzip on;
        gzip_types text/plain text/css application/json application/javascript text/xml;
        gzip_min_length 1000;
    }
}
```

## 7. CI/CD 流水线

### 7.1 GitHub Actions 配置

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  # ─── 后端测试 ───
  backend-test:
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: test
          MYSQL_DATABASE: power_load
        ports:
          - 3306:3306
        options: >-
          --health-cmd "mysqladmin ping -h localhost"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
      - name: Run tests
        run: cd backend && mvn test
      - name: Upload test report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: backend-test-report
          path: backend/target/surefire-reports/

  # ─── 前端测试 ───
  frontend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install & Lint & Test
        run: |
          cd frontend
          npm ci
          npm run lint
          npx vitest run --coverage
      - name: Upload coverage
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: frontend-coverage
          path: frontend/coverage/

  # ─── 构建检查 ───
  build-check:
    needs: [backend-test, frontend-test]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build backend JAR
        run: cd backend && mvn package -DskipTests
      - name: Build frontend
        run: |
          cd frontend
          npm ci
          npm run build
      - name: Verify build output
        run: |
          test -f backend/target/*.jar && echo "✅ Backend JAR OK"
          test -d frontend/dist && echo "✅ Frontend dist OK"
```

### 7.2 流水线触发条件

| 事件 | 执行内容 |
|:-----|:---------|
| PR → `develop` / `main` | 全量 test + lint + build |
| Push → `develop` | 全量 test + lint + build |
| Push → `feature/*` | 仅 lint（快速反馈） |

| 端点 | 用途 | 频率 |

## 10. 故障处理预案

### 10.1 常见问题速查

| 现象 | 可能原因 | 排查命令 | 解决方案 |
|:-----|:---------|:---------|:---------|
| 502 Bad Gateway | Backend 未就绪 | `docker-compose logs backend` | 等待 health check 通过 |
| 预测接口 500 | ML 推理服务挂了 | `curl http://localhost:5000/health` | `docker-compose restart ml-inference` |
| Agent 无响应 | LLM API 超时 / Key 错误 | `docker-compose logs backend \| grep LLM` | 检查 `.env` 中 `LLM_API_KEY` |
| 大屏数据不刷新 | WebSocket 未连接 | 浏览器 F12 → Network → WS | 检查 Nginx `/ws/` 代理配置 |
| MySQL 连接拒绝 | MySQL 未 ready | `docker-compose logs mysql` | 等待 health check 通过（最长 30s） |
| Flyway 迁移失败 | SQL 脚本有误 / 重复执行 | `docker-compose logs backend \| grep Flyway` | 检查迁移文件命名 |

### 10.2 快速回退

```bash
# 回退到上一个稳定版本
git checkout <last-stable-commit>
docker-compose down
docker-compose up -d --build
```

### 10.3 演示备用方案

| 故障 | 备用方案 |
|:-----|:---------|
| Docker 整个环境起不来 | 本地 `mvn spring-boot:run` + `npm run dev` + `python app.py` |
| LLM API 不可用 | Agent 降级为规则查询（不调 LLM，MySQL 直接查） |
| ML 推理不可用 | 展示历史预测数据（数据库已有） |
| 网络中断 | 本地 localhost 全栈演示（零外部依赖，除 LLM 外） |
